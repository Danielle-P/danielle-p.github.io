MealsUp

The Idea & Specification

The idea for MealsUp originated from our intention to become more organised in planning meals, to reduce waste and to make scheduling healthy and interesting meals easy in the context of a busy lifestyle. We thought these were challenges many people faced.

The aim was therefore to create an easy to use IOS App for scheduling meals. We also wanted to incorporate automatic generation of shopping lists based on the meals scheduled.

We identified three fundamental components for the App:

  • List of Dishes - the ability to create and save ‘Dishes’, with an accompanying list of ingredients, including quantities, and an optional link to an online recipe.
  • Meals schedule or ‘Menu’ - a simple list view of Dishes scheduled on each day, broken down into:
  • Mealtime (the names of which are customisable), e.g. breakfast, lunch, dinner
  • Course - starter, main, side 1, side 2 and dessert
  • Groceries list - a list of ingredients automatically added to a groceries/shopping list when a Dish is scheduled, plus any which are manually added by the user. Groceries are split between:
  • Groceries tagged as ‘need to be purchased in the next shopping trip’
  • Groceries which are in the list, but which the user has not tagged as ‘next shop’, i.e. they are needed but will need to be purchased at some time in the future (e.g. for a meal scheduled in a week’s time).
  • Groceries which have been marked as purchased/ticked off the list for reference so that other members of the household can see what has already been purchased.
  • The ability to share all of the above in real time with others in the same household who also have the App.

User Story

A typical user story would be:

Sign up

  • Create a home and invite other members of the household to that home

Everyday use

  • Create a dish in the Dishes list.
  • Schedule a dish for a specific date and for a specific mealtime in the Menu tab.
  • Add any custom ingredients to the groceries list in addition to those automatically generated by adding a dish to the Menu.
  • Mark as ‘next shop’ any groceries needed and tick off those which are not required (i.e. those which the user already has at home).
  • In the supermarket, tick off the groceries as they are purchased.

The Design

I was responsible for the design of MealsUp. The design decisions I made were based on replacing the process of physically writing a menu for the next week and writing a shopping list based on what the dishes/recipes required. This approach helped to keep the App simple and easy to use, and ensured the functionality was intuitive and familiar.

Features we added

We initially worked on the three main features which were mocked up in the initial wireframes, with a view to getting the MVP on the App Store as soon as possible. As soon as we had done that, we began to work on the ideas for extra features that we had identified during the design process and during the course of using and testing the App.

Additional features we added include:

  • Reminder function for the groceries list, reminding the user to buy the items on the next shop
  • Notifications when other members of the household have scheduled meals, or ticked off items on the groceries list
  • A ‘Surprise Me’ function to randomly generate an entire meal or a specific course from the user's list of dishes in the App
  • Basic machine learning engine to recommend ingredients when a user creates a dish, based on its name
  • Images of dishes derived from the link to the dish’s online recipe
  • An optional homepage dashboard view showing:
  • Most- and least-often scheduled dishes in the last 3 months
  • A snapshot of the upcoming Menu
  • A snapshot of the current groceries list
  • Method view

Testimonials

A Use Case

One of our users is a care home worker in the US, who uses MealsUp to schedule meals for the residents at the care home she works at and to manage the ordering and purchase of ingredients in specific quantities each week. This user requested the option to print the menu so that it could be displayed on a board within the care home, which was particularly important as some of the residents had allergies or were on specialised diets. We therefore added this feature in the Menu section of the App.
